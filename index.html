display:flex 布局详解

Posted on 2018-05-08

原文链接：https://zhuanlan.zhihu.com/p/25303493

flex 基本概念

在 flex 容器中默认存在两条轴，水平主轴(main axis) 和垂直的交叉轴(cross axis)，这是默认的设置，当然你可以通过修改使垂直方向变为主轴，水平方向变为交叉轴，这个我们后面再说。

在容器中的每个单元块被称之为 flex item，每个项目占据的主轴空间为 (main size), 占据的交叉轴的空间为 (cross size)。

这里需要强调，不能先入为主认为宽度就是 main size，高度就是 cross size，这个还要取决于你主轴的方向，如果你垂直方向是主轴，那么项目的高度就是 main size.

<!-- more -->

flex容器

首先，实现 flex 布局需要先指定一个容器，任何一个容器都可以被指定为 flex 布局，这样容器内部的元素就可以使用 flex 来进行布局。

默认值：row，主轴为水平方向，起点在左端。

row-reverse：主轴为水平方向，起点在右端

小程序之渲染机制

Posted on 2018-05-05

  1. 小程序渲染机制

    • 界面由成熟的Web技术渲染
    • 每个小程序页面都用不同的WebView进行渲染
    • 双线程模型 提供一个沙箱，分为逻辑层、渲染层
    • 渲染层只与数据相关，逻辑层通过setData函数设置数据，在界面渲染完成后触发setData
    • 每次渲染只需要把改变的数据渲染，没有改变的不渲染

  2. 页面初始数据通信

    • 页面初始数据（data)和路径等相关信息会从逻辑层发送给渲染层，用于渲染层的初始渲染
    • Native层会将这些数据直接传递给渲染层，同时向用户展示一个新的页面层级，渲染层在这个页面上进行界面绘制
    • 渲染层接受到相关数据后，根据页面路径来选择合适的WXML结构，WXML结构与初始数据相结合，得到第一次渲染的结果
    • 页面初始时间：大致由初始数据通信时间和初始渲染时间两部分构成。
    • 数据通信时间：指数据从逻辑层开始组织数据到渲染层完全接收完毕的时间
    • 更新数据通信 初始渲染完毕后，渲染层在开发者调用setData后执行界面更新
    • 在数据传输时候逻辑层会执行一次JSON.stringify来去掉setData数据中不可传输的部分

  3. 更新数据通信优化

    • 不要过度频繁调用setData，应该将多次setData合并成一次setData调用
    • 与界面渲染无关的数据最好不要设置在data中，可以考虑设置在page对象的其他字段下

  4. 数据驱动

    • 将WXML转换为JS对象
    • 对比前后两个JS对象变化的部分，将差异应用到原来的DOM树上，从而达到更新UI的目的
  1. ##应用构造器App()
    • 小程序宿主环境提供了App()构造器来注册一个应用
    • App()必须写在项目根目录的app.js文件中
    • App()实例是单例对象，不能注册多个！其他脚本内可以通过getApp()获取这个对象
    • onLaunch（小程序初始化完成后触发）、onShow（小程序从后台显示的时候触发）、onHide(小程序进入后台的时候触发)、OnError

  2. 全部数据globaldata，因为App()是单例的，所以可以通过设置App的数据来设置全局数据

  3. 应用声明周期

    • 进入小程序->触发onLaunch事件
    • 关闭或者按下Home后->触发onHide事件
    • 再次打开小程序的时候 ->触发onShow事件

  4. scene和referrerInfo

    • 小程序有不同的场景值
    • Android系统通过Home返回桌面再次进入小程序的场景值为上一次的场景值

  5. 页面构造器Page()

    • onLoad onReady onShow onHide onUpload生命周期函数
    • onPullDownRefresh onReachBottom onShareAppMessage onPageScroll页面的用户行为回调函数
  6. ##setData注意事项
    • 直接修改this.data而不调用this.setData是无法改变页面的状态的，还会造成数据不一致
    • 单词设置的数据不能超过1024kb，请尽量避免一次设置过多的数据
    • 不要把data中任何一项的value设置为undefined，否则不会渲染留下潜在问题

小程序组件

Posted on 2018-05-05

  1. view视图容器

    • 最基础的小程序组件，借助这个进行可以划分，页面调整
    • 最基础的组件只有四个属性：hover-class、hover-stop-propagation、hover-start-time 、 hover-stay-time
    • hover-class ： 点击下去的类样式 <view hover-class="xx"> 点击使用使用xx样式
    • hover-stop-propagation : 阻止点击态，类型为Boolean类型，禁止冒泡 更多信息可以百度HTML 元素事件冒泡
    • hover-start-time : 点击后开始的时间 ，时间单位为毫秒
    • hover-stay-time : 手指松开后点击态保留的时间

  2. scroll-view滚动视图容器

    • 能够在页面形成滚动展示效果，滑动展示
    • 属性： scroll-x scroll-y 两个属性均为Boolean类型，为true为支持x或y上的滑动
    • 属性： upper-threhold low-threshold 类型均为Number 默认值为50 滑动到距离顶部/左边、底部/右边多远的时候触发，单位为px，这个功能可以用于滑动协议内容到底部后才能同意协议 或者 滑动到底部后自动切换到下一章内容
    • bindscrolltoupper bindscrolltolower 滑动到顶部和底部的时候触发的事件 类型为EventHandler
    • textarea map canvas video 这些组件不能在scroll-view使用
    • 在滚动scroll-view的使用会阻止页面回弹，所以在scroll-view中滚动是无法触发onpulldownrefresh
    • 若要使用下拉刷新，请使用页面的滚动，而不是scroll-view，这样也能通过点击顶部状态栏回到页面顶部

  3. movable-view 可移动视图容器

    • 能够实现拖动
    • 该组件能够缩放
    • 该组件能监控手指初次触摸和移动方向
    • movable-view必须在<movable-area>组件中，并且为直接子节点，否则不能移动
    • movable-view必须设置width和height属性，不设置默认为10px
    • movable-view默认为绝对定位，top left均为0
    • 使用场景：精细动态交互的时候使用
  4. cover-view 覆盖文字组件、cover-image在原生组件上展示出一些图片内容
    • 能够覆盖在原生组件上，从而实现在原生组件上显示一些文本内容
    • 可以覆盖的组件：map、video、canvas、camera
  5. text组件只能嵌套text组件；rich-text 中nodes属性推荐使用Array类型，因为组件还是会将String类型转换为Array类型

小程序之JavaScript

Posted on 2018-05-05

  1. 小程序JavaScript和EMCAScript

    • JavaScript是对EMCAScript的一种实现
    • 小程序JavaScript的框架示意： EMCAScript --> 小程序框架 --> 小程序API
    • 浏览器JavaScript的框架示意：EMCAScript --> DOM --> BOM
    • Nodejs的JavaScript框架示意：EMCAScript --> NPM --> Native
    • 小程序没有DOM所以依赖DOM的JQuery无法使用
    • IOS9和IOS10 不支持ES6，所以需要在IDE中将ES6语法转换为ES5语法
  2. ##Commonjs模块化
    • 小程序内部使用commonjs来进行模块的管理和处理，所以需要遵循commonjs规范
    • commonjs使用require方法加载模块
    • 模块加载是同步的，只有加载完成后才能执行后面的操作
    • module.exports属性是当前模块对外的接口，调用模块实际上是调用module.export的变量
